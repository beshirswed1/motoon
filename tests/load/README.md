# دليل اختبار الأداء والحمل الشامل باستخدام k6 (MOTOON Load Testing)

يحتوي هذا المجلد على كود اختبار الأداء والتحمل لمنصة **متون** باستخدام أداة **k6**.

يقوم هذا الاختبار بمحاكاة سلوك حقيقي متكامل للمستخدمين عبر تصفح **كافة واجهات ومسارات المستخدم (مع استثناء لوحة التحكم الخاصة بالمسؤولين/المدراء admin)**.

---

## 1. مسارات الفحص المشمولة (Tested Routes)

ينقسم الاختبار إلى مجموعات (Groups) تحاكي تدفق المستخدم الفعلي:

1. **الصفحات التسويقية والعامة (Public Pages):**
   * الصفحة الرئيسية (`/`)
   * صفحة من نحن (`/about`)
   * صفحة اتصل بنا (`/contact`)
   * سياسة الخصوصية (`/privacy`)
   * شروط الخدمة (`/terms`)
2. **صفحات تسجيل الدخول والتسجيل (Auth Pages):**
   * صفحة تسجيل الدخول والبدائل (`/login` و `/sign-in`)
   * صفحة التسجيل والبدائل (`/register` و `/sign-up`)
3. **قائمة المتون (Books Catalog):**
   * صفحة الفهرس الرئيسية للمتون (`/books`)
   * استعلام الـ API لجلب كافة المتون (`/api/books/all`)
4. **صفحات المتن الفردي (Single Book Details):**
   * صفحة تفاصيل المتن المختار عشوائياً (`/books/[slug]`)
   * استعلام الـ API لبيانات المتن (`/api/books/[slug]`)
   * صفحة القراءة (`/books/[slug]/read`)
5. **صفحات المذاكرة والدراسة (Study Modules):**
   * صفحة الحفظ والتكرار (`/books/[slug]/memorize`)
   * صفحة التسميع والمراجعة الصوتية (`/books/[slug]/recite`)
   * صفحة الشهادات للمتن (`/books/[slug]/certificate`)
6. **صفحات الحساب الشخصي (User Dashboard):**
   * الملف الشخصي للمستخدم (`/profile`)
   * لوحة التقدم الدراسي (`/progress`)
   * المتون المفضلة (`/favorites`)
   * الإشعارات (`/notifications`)
   * الإعدادات (`/settings`)
7. **صفحة الطوارئ غير المتصلة بالشبكة (PWA Offline Fallback):**
   * صفحة (`/offline`)

---

## 2. تثبيت k6 على نظام التشغيل Windows

افتح Terminal (PowerShell أو CMD) كمدير للجهاز (Administrator) واكتب:

```powershell
winget install GrafanaLabs.k6 --accept-source-agreements --accept-package-agreements
```

> **ملاحظة:** بعد انتهاء التثبيت، يجب إغلاق موجّه الأوامر وفتحه من جديد لتحديث مسارات البيئة (PATH).
> للتحقق من نجاح التثبيت اكتب:
> ```powershell
> k6 version
> ```

## 3. تشغيل الاختبارات (Running the Tests)

يُمكن تشغيل الفحص بعدة طرق وفقاً للمطلوب:

### أ. اختبار الفحص السريع للتأكد من السلامة (Smoke Test)
يستخدم مستخدماً واحداً (1 VU) لتشغيل دورة واحدة سريعة للتأكد من عمل كافة الواجهات وعدم وجود أخطاء صياغة.
```powershell
k6 run --vus 1 --duration 5s tests/load/k6-load-test.js
```

### ب. اختبار الحمل الطبيعي التدريجي (Load Test)
يرتفع تدريجياً لـ 20 مستخدماً وهمياً لمدة دقيقة ثم يهبط.
```powershell
k6 run tests/load/k6-load-test.js
```

### ج. اختبار الضغط العالي (Stress Test)
يقوم بالضغط على الخادم بـ 100 مستخدم وهمي لمدة دقيقتين لقياس سلوك النظام تحت عبء شديد.
```powershell
k6 run --vus 100 --duration 2m tests/load/k6-load-test.js
```

### د. فحص خادم بعيد أو خادم تجربة (Staging / Production)
بشكل افتراضي، يستهدف الاختبار الرابط المحلي `http://localhost:3000`. لتغيير الرابط المستهدف:
```powershell
k6 run -e BASE_URL=https://staging.motoon.com.tr tests/load/k6-load-test.js
```

## 4. قراءة النتائج وفهم المقاييس (Understanding the Output)

بعد انتهاء الفحص، ستظهر إحصائيات مقسمة حسب المجموعات (Groups) المذكورة أعلاه. يرجى مراجعة:
* **`checks`**: نسبة نجاح التحقق من استجابة الصفحات (الهدف: 100%).
* **`http_req_failed`**: نسبة فشل طلبات الـ HTTP (الهدف: أقل من 1%).
* **`http_req_duration`**: زمن استجابة الخادم بالمللي ثانية (الهدف: 95% من الطلبات أقل من 1500ms).
