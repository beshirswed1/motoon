# دليل نشر وإطلاق منصة متون (MOTOON Deployment Guide)

يركز هذا الدليل على خطوات النشر والتهيئة النهائية لمنصة **متون** على منصة Vercel السحابية وتهيئة قواعد بيانات Firebase Firestore.

---

## 1. إعداد متغيرات البيئة (Environment Variables)

عند تهيئة المشروع في لوحة تحكم Vercel، يجب إضافة المتغيرات التالية بدقة وتطابق تام:

| المتغير | الوصف | مثال / القيمة |
| :--- | :--- | :--- |
| `NEXT_PUBLIC_APP_URL` | الرابط الرسمي للمنصة (بدون شرطة مائلة في النهاية) | `https://motoon.sa` |
| `NEXT_PUBLIC_APP_NAME` | اسم التطبيق المعروض للمستخدمين | `متون` |
| `NEXT_PUBLIC_FIREBASE_API_KEY` | مفتاح واجهة برمجة التطبيقات لـ Firebase | `AIzaSyA1...` |
| `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN` | نطاق المصادقة لمشروع فايربيز | `motoon-app.firebaseapp.com` |
| `NEXT_PUBLIC_FIREBASE_PROJECT_ID` | معرّف مشروع فايربيز | `motoon-app` |
| `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET` | مساحة التخزين لملفات الصوت والصور | `motoon-app.appspot.com` |
| `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID` | معرّف مرسل الإشعارات | `847592...` |
| `NEXT_PUBLIC_FIREBASE_APP_ID` | معرّف تطبيق الويب في فايربيز | `1:847592:web:...` |
| `NEXT_PUBLIC_IMGBB_API_KEY` | مفتاح ImgBB لرفع أغلفة الكتب والمتون | `your_imgbb_key` |

---

## 2. إعداد فهارس Firestore المركبة (Firestore Composite Indexes)

تتطلب بعض استعلامات الاسترجاع والترتيب فهارس مركبة لتجنب توقف الاستعلامات. تم توفير الفهارس اللازمة في ملف `firestore.indexes.json` في جذر المشروع.

لتطبيق الفهارس مباشرة باستخدام Firebase CLI:
```bash
firebase deploy --only firestore:indexes
```

أو يمكنك إنشاؤها يدوياً في لوحة تحكم Firebase Console كالتالي:

1. **مجموعة progress**:
   - الحقول: `userId` (تصاعدي) و `nextReviewDate` (تصاعدي).
2. **مجموعة recitation_sessions**:
   - الحقول: `userId` (تصاعدي) و `createdAt` (تنازلي).
3. **مجموعة books**:
   - الحقول: `isDeleted` (تصاعدي) و `isPublished` (تصاعدي) و `createdAt` (تنازلي).
4. **مجموعة activity_logs**:
   - الحقول: `performedBy` (تصاعدي) و `timestamp` (تنازلي).

---

## 3. قواعد الحماية لـ Firestore (Security Rules)

تأكد من تطبيق قواعد الحماية المرفقة في ملف `firestore.rules` لحماية خصوصية بيانات الطلاب وضمان صلاحيات التعديل للمشرفين فقط:
```bash
firebase deploy --only firestore:rules
```

---

## 4. تهيئة الحساب الإداري الأول (Super Admin Setup)

عند تشغيل التطبيق لأول مرة:
1. قم بإنشاء حساب جديد عبر صفحة تسجيل الحساب `/register`.
2. افتح بوابة التطوير المصغرة (الزر العائم الأحمر في أسفل اليسار - متاح فقط في البيئة المحلية Localhost أو عبر المحاكاة).
3. اضغط على خيار **مدير (Admin)** ليتم تحديث صلاحية حسابك الحالي فوراً في Firestore ومزامنة الكوكيز.
4. بمجرد الترقية، ستتمكن من تصفح لوحة تحكم الإدارة بالكامل عبر الرابط `/admin`.

---

## 5. خطوات النشر على Vercel (Vercel Deployment Steps)

1. **ربط المستودع:** قم بربط مستودع Git الخاص بالمشروع بحسابك في Vercel.
2. **إعدادات البناء:** يكتشف Vercel تلقائياً إعدادات Next.js بشكل افتراضي.
   - Framework Preset: `Next.js`
   - Build Command: `next build`
   - Output Directory: `.next`
3. **إضافة متغيرات البيئة:** انسخ القيم من ملفك المحلي `.env.local` والصقها في إعدادات البيئة بـ Vercel.
4. **النشر:** اضغط على **Deploy**. سيقوم Vercel ببناء المشروع وتوفير رابط معاينة آمن وفعال.

---

## 6. قائمة التدقيق قبل الإطلاق النهائي (Pre-launch QA Checklist)

تأكد من فحص التدفقات التالية قبل الإعلان عن إطلاق المنصة رسمياً للجمهور:

### أ. فحص الحساب الإداري (Admin Audit)
- [ ] الدخول إلى لوحة تحكم الإدارة `/admin` والتحقق من عمل الرسوم البيانية لإحصائيات المستخدمين.
- [ ] فحص نظام إدراج وتجزئة الأبيات (Verse Paste Parser) عن طريق نسخ ولصق متن طويل (أكثر من 100 بيت) والتأكد من تجزئتها وحفظها بشكل صحيح.
- [ ] فحص إمكانية تعديل وحذف المتون والأبيات وتسجيل العمليات في سجل الرقابة (Audit Logs).

### ب. فحص حساب طالب العلم (Student Flow Audit)
- [ ] تصفح قائمة المتون العلمية المتاحة وتجربة محرك التكرار المتباعد (Spaced Repetition) لحفظ وضبط متن معين.
- [ ] تشغيل تدفق التسميع الصوتي (Recitation Flow) بالكامل: تسجيل الصوت، تتبع تمييز الكلمات المقروءة، وتوليد الدرجات وتقييم أحكام التجويد والطلاقة وعرض النتيجة بشكل واضح ومبسط باللغة العربية.
- [ ] التحقق من عمل نظام التنبيهات وإرسال الإشعارات عند حلول موعد المراجعة القادمة للمتون.

### ج. تدقيق الوصول والخصوصية (Security & Access Check)
- [ ] محاولة الدخول إلى الرابط `/admin` باستخدام حساب طالب عادي والتأكد من إعادة التوجيه الفوري للرئيسية `/`.
- [ ] التأكد من تشفير وحماية حقول كلمات المرور والبيانات الحساسة في الـ Cookies والمخازن المحلية.
- [ ] التحقق من عدم وجود أي كلمات مرور أو مفاتيح سرية Firebase/ImgBB مكشوفة في الكود المصدري للعميل.

### د. فحص تطبيق الويب التقدمي (PWA Checklist)
- [ ] التحقق من إمكانية تثبيت التطبيق كـ PWA على نظام أندرويد (تلقائياً عبر المتصفح).
- [ ] التحقق من عمل موجه التثبيت اليدوي المخصص لـ iOS (تعليمات Safari المرفقة في البانر السفلي).
- [ ] فحص إمكانية التصفح والمراجعة في الوضع دون اتصال (Offline Mode) بشكل كامل ومزامنة التقدم تلقائياً بمجرد استعادة الاتصال.
